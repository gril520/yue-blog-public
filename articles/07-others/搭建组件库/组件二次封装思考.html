组件封装思考 ​

一、封装理解 ​

我们在日常工作中，会基于第三方UI组件库来封装成为自己需要的组件库，比如：element-ui、vant-ui、iview、ant-design等等。不管是为了业务考虑还是单纯的为了提高效率，我们会把一些经常用到的组件抽离、封装成公共组件，这样方便我们在不同的地方使用这个组件，减少重复代码的编写。

我们把对于第三方组件库的封装称为组件的二次封装，那么这带来有个思考，当我们在二次封装时，我们在封装什么？

在 vue 组件封装时，我们需要注意的主要是三部分：prop、event、slot。

  • prop：表示组件接收的参数，最好用对象的写法，这样可以针对每个属性设置类型、默认值或自定义校验属性的值，此外还可以通过type、validator等方式对输入进行验证；
  • event：子组件向父组件传递消息的重要途径；
  • slot：可以给组件动态插入一些内容或组件，是实现高阶组件的重要途径；当需要多个插槽时，可以使用具名slot。

二、多级组件数据传递 ​

我们多级组件嵌套需要传递数据时，通常使用的方法是通过vuex。如果仅仅是传递数据，而不做中间处理，使用 vuex 处理，这就有点大材小用了。所以就有了 $attrs / $listeners ，通常配合 inheritAttrs 一起使用。

inheritAttrs 是一个布尔属性（默认为 true），用于确定是否要将父组件中未被识别的 (即没有在组件的 props 中声明的) attribute 绑定 (attribute bindings) 自动绑定到子组件的根元素上。

当你设置为 false 时，这些 attribute 将不会出现在子组件的根元素上，但你可以通过 v-bind="$attrs" 显式地将它们绑定到子组件内部的某个元素上。

vue
<template>  
  <div v-bind="$attrs">  
    <!-- 内部内容 -->  
    <slot></slot>  
  </div>  
</template>  
  
<script>  
export default {  
  inheritAttrs: false,  
  mounted() {  
    // 你可以在这里访问 $attrs 和 $listeners  
    console.log(this.$attrs);  
    console.log(this.$listeners);  
  }  
}  
</script>

传递属性 ​

在 Vue.js 中，$attrs 是一个特殊的属性，它包含了父组件中未被子组件显式声明为 props 的所有属性（attribute）。这些属性会作为一个对象传递给子组件，并且可以通过 v-bind 或简写 : 与子组件的根元素进行绑定。

这里有几个关键点：

  1. 不包含 class 和 style：即使父组件为子组件提供了 class 和 style 属性，这些属性也不会包含在 $attrs 中，因为它们有特殊的处理方式。
  2. 未被声明为 props：只有那些没有在子组件中通过 props 选项明确声明的属性才会被包含在 $attrs 中。
  3. 传递方式：你可以通过 v-bind="$attrs" 将 $attrs 传递给子组件，这样，子组件的根元素就会获得这些未被声明的属性。
  4. 默认行为：如果一个组件没有声明任何 props，那么 $attrs 将包含所有从父组件传递下来的属性（除了 class 和 style）。

示例，假设你有以下的组件结构：

vue
<!-- ParentComponent.vue -->  
<template>  
  <ChildComponent foo="bar" baz="qux" class="parent-class" style="color: red;"></ChildComponent>  
</template>  
  
<script>  
import ChildComponent from './ChildComponent.vue';  
  
export default {  
  components: {  
    ChildComponent  
  }  
}  
</script>

传递事件 ​

在 Vue.js 中，$listeners 是一个特殊的属性，它包含了父作用域中（不含 .native 修饰符）的 v-on 事件监听器。这意味着，如果你在一个组件上定义了一些事件监听器（如 @click、@submit 等），并且这个组件没有在其内部显式地声明或处理这些事件，那么这些事件监听器就会被包含在 $listeners 对象中。

这个特性特别有用，尤其是在创建可重用的组件时，因为你可以让子组件“继承”父组件的事件监听器，而无需在子组件中显式地声明或转发这些事件。

示例，假设我们有一个父组件 ParentComponent 和一个子组件 ChildComponent：

vue
<!-- ParentComponent.vue -->  
<template>  
  <ChildComponent @click="handleClick" @custom-event="handleCustomEvent" />  
</template>  
  
<script>  
import ChildComponent from './ChildComponent.vue';  
  
export default {  
  components: {  
    ChildComponent  
  },  
  methods: {  
    handleClick() {  
      console.log('Clicked from parent');  
    },  
    handleCustomEvent() {  
      console.log('Custom event triggered from parent');  
    }  
  }  
}  
</script>

注意

  • $listeners 不包含带有 .native 修饰符的事件监听器，因为这些监听器是直接绑定到子组件的根元素上的，而不是作为组件的事件来处理的。
  • 当你使用 v-on="$listeners" 时，你实际上是在将父组件上的所有事件监听器都绑定到了子组件的根元素上。这可能导致一些意外的行为，特别是当子组件有自己的事件处理逻辑时。因此，在使用 $listeners 时要小心，确保它符合你的预期行为。

如何使用 $attrs 和 $listeners

在使用 el-input-number时，当我们给他赋默认值 null 或者空字符串 "" 时，会显示 0 ，而这在我们一些业务场景里并不是很友好，并且值是居中显示的，那么现在我们想要做的改造是：值居左显示，没有默认值显示0的问题，且默认不展示控制按钮

  • 控制按钮默认不显示：controls 设置成 false
  • 居左显示：通过样式控制
  • 默认值显示0的问题：通过 computed 计算不为 number 类型时，赋值为 undefined 解决

v-model 是一个语法糖，可以拆解为 props: value 和 events: input。就是说组件只要提供一个名为 value 的 prop，以及名为 input 的自定义事件

经过对 el-input-number 二次封装，我们的需求其实已经基本实现了，但是我们希望其用法保持和 el-input-number 组件相似，这样即使其他人在使用我们封装的组件时，也能参照 element 对应的文档正常将其使用起来。

二次封装尽量遵循的应该是原有基础的扩展，不管是为了针对业务还是为了方便使用，而不是为组件重新制定一套新的用法，毕竟封装的本质是为了提高使用体验，而不是增加更多不必要的心智负担。

透传分析 ​

Vue.js 中通过 v-model 指令以及 $attrs 和 $listeners 的传递机制实现跨组件的“双向绑定”效果，特别是当涉及到多层嵌套组件时。这种机制使得我们可以在不直接操作子组件内部状态的情况下，通过 v-model 在父组件和子组件（或更深层的后代组件）之间建立一种类似双向绑定的通信方式。

具体来说：

  1. attrs的传递：在Vue2.x中，如果父组件向子组件传递了属性（attributes），但子组件没有声明这些属性作为props，那么这些属性会被自动绑定到子组件的根元素上。但是，如果子组件使用了‘inheritAttrs:fase*‘，这些未声明的属性则不会绑定到根元素上，而是可以通过‘v−bind*="attrs"显式地将它们绑定到子组件内部的某个元素或子组件上。Vue 3.x 中对$attrs 的处理略有不同，但基本概念相似。
  2. liste*ners*的传递：listeners包含了父组件中（不含.native修饰符）的v-on事件监听器。子组件可以通过v-on="$listeners"将这些事件监听器传递给其内部的元素或子组件，从而实现了事件的冒泡或传递。
  3. v-model 的工作原理：v-model 实际上是一个语法糖，用于简化表单输入和组件之间的双向绑定。在组件上，v-model 通常会监听一个名为 input 的自定义事件，并更新与 value prop 绑定的数据。因此，当我们在父组件中使用 v-model 绑定一个子组件时，Vue 会自动将 value 作为 prop 传递给子组件，并监听子组件发出的 input 事件来更新父组件的数据。

在多层嵌套组件的场景中，如果中间层组件正确地传递了 $attrs 和 $listeners，那么最底层的组件就可以通过发出 input 事件来影响父组件的数据，而父组件也可以通过改变绑定的数据来影响最底层组件的显示。这种机制使得我们可以在不直接操作子组件内部状态的情况下，实现跨组件的“双向绑定”效果。

需要注意的是，虽然这种机制在某些场景下非常有用，但它也可能导致组件之间的耦合度过高，使得代码难以维护和扩展。因此，在实际开发中，我们应该谨慎使用 v-model 和 $attrs/$listeners 的传递机制，并根据具体需求选择合适的通信方式。

五、插槽 ​

前面所封装组件已经继承了 el-input 的所有属性及事件，但是 el-input 为了更方便用户自定义还提供了一系列插槽，所以我们的封装也应该继承这些插槽。

要点：

elment ui组件带有插槽功能，如何在二次封装后的组件也支持其本身就带有的功能呢？

普通插槽——Slots ​

在 Vue 3 中，使用 <template> 标签配合 v-slot 指令或者新的 #slotName 语法（这是 Vue 3 的简写语法）来定义插槽内容是一种常见的做法。

在 Vue 组件中，我们不会在 <template> 标签内部使用 <slot> 标签来定义插槽，相反，我们会在组件的模板中预留一个位置，这个位置就是插槽。然后，在父组件中使用这个组件时，我们可以通过具名插槽（使用 name 属性）或者默认插槽来传递内容到这个位置。

以下是一个简单的示例，说明如何在 Vue 3 组件中定义和使用插槽：

子组件（定义插槽）

vue
<template>  
  <div>  
    <!-- 这里预留了一个插槽位置 -->  
    <slot name="slotName"></slot>  
    <!-- 也可以预留一个默认插槽位置 -->  
    <slot></slot>  
  </div>  
</template>  
  
<script>  
export default {  
  name: 'ChildComponent',  
  // ... 其他选项  
}  
</script>

动态插槽——$Slots ​

在 Vue 中，如果你需要传递的插槽名称不固定或者较多，可以使用动态插槽名称来实现。在 Vue 2 和 Vue 3 中，虽然语法有所不同，但核心思想是一致的：通过动态绑定来指定插槽的名称。

vue
<template #[slotName] v-for="(slot, slotName) in $slots" >
    <slot :name="slotName" />
</template>

作用域插槽——Scoped Slots ​

在 Element UI 中，作用域插槽（Scoped Slots）是一种特殊的插槽，它允许你访问子组件的数据。这些数据通常是作为一个对象通过插槽的 slot-scope 或 v-slot 指令传递的。

在 Vue 2.6.0+ 中，推荐使用 v-slot 语法来替代 slot-scope。

指令推荐都使用缩写形式，(用 : 表示 v-bind: 、用 @ 表示 v-on: 和用 # 表示 v-slot:)

以下是一个使用 Element UI 表格组件（el-table）和作用域插槽的示例：

Vue 2.x 示例 (使用 slot-scope)

vue
<template>  
  <el-table  
    :data="tableData"  
    style="width: 100%">  
    <el-table-column  
      prop="date"  
      label="日期"  
      width="180">  
    </el-table-column>  
    <el-table-column  
      label="操作"  
      width="120">  
      <template slot-scope="scope">  
        <el-button @click="handleClick(scope.row)">点击我</el-button>  
      </template>  
    </el-table-column>  
  </el-table>  
</template>  
  
<script>  
export default {  
  data() {  
    return {  
      tableData: [{  
        date: '2023-06-20',  
        name: '张三',  
        // ... 其他数据  
      }, /* 更多数据 */]  
    };  
  },  
  methods: {  
    handleClick(row) {  
      console.log(row);  
    }  
  }  
};  
</script>

$scopedSlots 是 Vue.js 中的一个属性，主要用于访问组件的作用域插槽。在 Vue.js 中，插槽（slots）是一种允许父组件向子组件中插入内容的机制。而作用域插槽（scoped slots）则是一种特殊的插槽，允许父组件访问子组件中的数据。

具体来说，$scopedSlots 是一个对象，其键是插槽的名字，值是一个函数，这个函数接收一个参数（通常是插槽的作用域对象），并返回一个 VNode（虚拟节点）数组或 undefined。这个对象在使用渲染函数（render function）开发 Vue 组件时特别有用，因为它允许你在渲染过程中动态地决定要渲染哪些插槽，以及这些插槽如何根据子组件的作用域数据来渲染。

下面是一个简单的例子来说明 $scopedSlots 的用法：

  </div>  
</template>  
  
<script>  
export default {  
  data() {  
    return {  
      item: { name: 'Item 1', value: 'Value 1' }  
    }  
  }  
}  
</script>

可选链和空值合并 ​

在JavaScript（特别是现代JavaScript，如ES2020及以后版本）中，config?.selection??.show 这样的表达式是一个组合使用了可选链（Optional Chaining）和空值合并（Nullish Coalescing）的表达式。

  1. 可选链（Optional Chaining）：?. 操作符用于读取位于连接对象链深处的属性的值，而不必明确验证链中的每个引用是否有效。如果尝试访问的某个属性不存在或值为null/undefined，表达式会立即返回undefined，而不是抛出一个错误。

例如：

javascript
const config = {  
  selection: {  
    show: true  
  }  
};  
  
const show = config?.selection?.show; // show 将会是 true  
  
const anotherConfig = {  
  // selection 属性不存在  
};  
  
const anotherShow = anotherConfig?.selection?.show; // anotherShow 将会是 undefined
1

  1. 空值合并（Nullish Coalescing）：?? 操作符是一个逻辑操作符，当左侧的操作数为 null 或 undefined 时，返回其右侧的操作数，否则返回左侧的操作数。但是，?? 操作符只会在 null 或 undefined 时才返回其右侧的值，而逻辑或 || 会在左侧操作数为假值时（如 0、''、NaN、false、null、undefined）返回右侧的值。

然而，在你给出的表达式 config?.selection??.show 中，空值合并操作符 ?? 的使用似乎是不正确的，因为它后面紧跟着 .show，这会导致语法错误。空值合并操作符通常用于直接赋值或返回值，而不是链式调用。

如果你想在 config?.selection 为 null 或 undefined 时提供一个默认值（比如一个空对象），并继续访问 .show 属性，你可以这样写：

javascript
const show = (config?.selection ?? {}).show;
1

这样，如果 config?.selection 是 null 或 undefined，那么 config?.selection ?? {} 就会返回一个空对象 {}，然后尝试访问 .show 属性将返回 undefined，而不是抛出一个错误。
