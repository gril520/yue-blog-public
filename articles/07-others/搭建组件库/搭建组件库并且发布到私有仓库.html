搭建组件库 ​

一、技术介绍 ​

使用的技术主要有 pnpm、vue、elment ui、eslint+prettier+husky

pnpm ​

pnpm 是 performant npm（高性能的 npm），它是一款快速的，节省磁盘空间的包管理工具，同时，它也较好地支持了 workspace 和 monorepo，简化开发者在多包组件开发下的复杂度和开发流程。

pnpm 为 performant npm 的简称，意为高性能的 npm

pnpm 主要有以下优点:

  • 快速: pnpm 比其他包管理工具快两倍;
  • 高效: node_modules 中的文件链接自特定的内容寻址存储库;
  • 支持 monorepo: pnpm 内置了对存储库中的多个包的支持;
  • 严格: pnpm 默认创建一个非平铺的 node_modules,因此代码不能访问任意包;

pnpm 内置了对单一存储库（也称为多包存储库、多项目存储库或单体存储库）的支持，只需要创建一个 [workspace] 就可将多个项目合并到一个仓库中，这样的作用是能在我们开发调试多包时，彼此间的依赖引用更加简单。

我们在根目录下新建 packages 文件夹，再新建 pnpm-workspace.yaml 文件，用来声明对应的工作区，写入如下内容：

yaml
packages:
  # 存放组件库和其他工具库
  - 'packages/*'
  # 存放组件测试的代码
  - 'example'

install` 函数是 Vue 插件的标准接口。它接受两个参数：`Vue`构造函数和`opts`选项对象（默认为空对象）。这个函数首先检查是否已经安装过该插件（通过`install.installed`属性），如果已经安装过则直接返回。然后，它遍历`components`数组，并使用`Vue.component`方法将每个组件注册为全局组件。每个组件的名称是从其`name` 属性中获取的。

这段代码是 JavaScript（或 TypeScript）模块导入语句，主要用于配置一个基于 Vite 的项目，特别是针对 Vue 2 的项目。我将为你逐行解释这些代码：

  1. import { fileURLToPath, URL } from 'node:url'
    • 这行代码从 Node.js 的 url 模块中导入了两个功能：fileURLToPath 和 URL。
    • fileURLToPath：将文件 URL 转换为路径字符串。这在处理 Node.js 中的 ES 模块路径时很有用。
    • URL：这是一个构造函数，用于创建 URL 对象。虽然在这段代码中只导入了它，但可能后续代码会用到它。
  2. import { defineConfig } from 'vite'
    • 从 vite 包中导入了 defineConfig 函数。Vite 是一个现代化的前端构建工具和开发服务器，用于快速开发 Vue、React 和其他框架的应用。
    • defineConfig 通常用于定义一个 Vite 配置对象，该对象包含构建和开发服务器的各种设置。
  3. // import legacy from '@vitejs/plugin-legacy'
    • 这是一个被注释掉的导入语句。如果取消注释，它将从 @vitejs/plugin-legacy 包中导入 legacy 插件。
    • plugin-legacy 是一个 Vite 插件，用于为旧版浏览器生成传统（legacy）构建。
  4. import vue2 from '@vitejs/plugin-vue2'
    • 从 @vitejs/plugin-vue2 包中导入了 vue2 插件。这是一个 Vite 插件，允许你在 Vite 项目中使用 Vue 2。
    • 当你想要将现有的 Vue 2 项目迁移到 Vite 或简单地使用 Vite 作为 Vue 2 的开发服务器和构建工具时，这个插件是非常有用的。
  5. import vue2Jsx from '@vitejs/plugin-vue2-jsx'

    • 这个插件可能是用于在 Vue 2 中使用 JSX 语法的。但请注意，Vue 通常推荐使用其模板语法，而不是 JSX，但如果你有更特定的需求或习惯使用 JSX，这个插件可能会有所帮助。
  6. import { resolve } from 'path'
    • 从 Node.js 的 path 模块中导入了 resolve 函数。
    • resolve 函数用于将路径或路径片段的序列解析为绝对路径。在配置 Vite 时，这通常用于指定项目中的文件或目录的路径。

总结：这段代码主要是为 Vite 项目导入所需的配置和插件，特别是针对 Vue 2 的项目。通过这些导入，开发者可以配置 Vite 以支持 Vue 2 的开发，并可能使用 JSX 语法（如果需要的话）。

这个 package.json 文件是 Node.js 项目（在这种情况下，很可能是一个 Vue 组件库或 UI 框架）的配置文件。以下是对给出的字段的解释：

  1. "name": "@yue/el-base-ui"
    • 这是包的名称。它通常用于在 npm（Node Package Manager）上唯一标识一个包。这里的名称以 @yue 开头，这通常表示它是一个组织（在这个例子中是 yue）的私有或公共包。
  2. "version": "0.0.1"
    • 这是包的版本号。在发布到 npm 或其他包管理器时，版本号用于标识包的特定版本。
  3. "description": "yue 组件集"
    • 这是包的简短描述。它应该简要地解释包的功能或用途。
  4. "private": false
    • 这个字段指示这个包是否是私有的。设置为 false 表示这是一个公开的包，可以被其他人从 npm 上安装。如果设置为 true，则这个包不能从 npm 上被直接安装（除非你有特殊的访问权限）。
  5. "main": "index.js"
    • 这是包的入口点，即当你 require 或 import 这个包时，Node.js 或某些打包工具（如 Webpack）会查找这个文件。但是，由于后面有 "type": "module" 和 "module" 字段，这个字段可能不是主要的入口点（特别是在 ES 模块或现代前端项目中）。
  6. "module": "./ui/es/index.js"
    • 当使用 ES 模块或现代前端打包工具（如 Rollup、Webpack 的某些配置）时，这个字段指定了包的入口点。在这个例子中，入口点位于 ui/es/index.js。
  7. "type": "module"
    • 这个字段告诉 Node.js 该包应该作为 ES 模块处理，而不是 CommonJS 模块。这意味着你可以在这个包中使用 import 和 export 语句，而不是 require 和 module.exports。
  8. "files": ["ui"]
    • 这个字段指定了当包被发布到 npm 时应该包括哪些文件或目录。在这个例子中，只有 ui 目录下的文件会被包括。这有助于确保不必要的文件（如测试文件、开发配置文件等）不会被意外地包含在 npm 包中。

总之，这个 package.json 文件描述了一个 Vue UI 组件库的基本信息、版本、描述、入口点以及发布到 npm 时应该包括的文件。
