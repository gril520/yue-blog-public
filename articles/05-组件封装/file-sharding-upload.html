TrFileShardingUpload 文件分片上传 ​

文件分片上传，适用于大文件上传场景。组件支持分片上传，断点续传；若文件过大，第一次未传输成功，再次选择该文件可续传。

目前组件支持minio和fastdfs上传。

基本用法 ​

基础的文件分片上传用法

::: demo

html
<template>
  <div>
    <file-sharding-upload
          :upload-percentage="uploadPercentage"
          :file-list="fileList"
          :is-uploading="isUploading"
          :is-auto-re-uploading="isAutoReUploading"
          @start-upload="handleStartUpload"
          @handle-remove="handleRemove"
        />
  </div>
</template>

<script>
import FileShardingUpload from './file-sharding-upload/file-upload.vue'
import { TrFileShardingUpload } from './file-sharding-upload/uploads.js'
import { delateFileByPath } from './file-sharding-upload/file-sharding-upload-api'

export default {
  name: 'index',
  components: {
    TrOnlyOffice,
    FileShardingUpload
  },
  data() {
    return {
      // Buckets 名称
      businessType: 'jxcc-common-file-dev',
      // 两种不同的分布式存储方案 minio | fastdfs
      saveType: 'minio',
      uploadPercentage: 0,
      fileList: [],
      isUploading: false,
      isAutoReUploading: false
    }
  },

  methods: {
    handleStartUpload(file) {
      // new TrFileShardingUpload
      const uploader = new TrFileShardingUpload(file, this.businessType, this.saveType)
      uploader.onProgress(progress => {
        // 处理进度更新事件
        console.log('上传进度：', progress + '%')
        this.uploadPercentage = progress
      })

      uploader.onFileInfo(fileInfo => {
        // 处理文件列表更新事件
        console.log('文件信息已更新：', fileInfo)
        this.fileList.push({
          ...fileInfo,
          url: fileInfo.fileUrl,
          name: fileInfo.fileName,
          uid: file.uid
        })
      })

      uploader.onIsUploading(isUploading => {
        console.log('isUploading：', isUploading)
        this.isUploading = isUploading
      })

      uploader.onIsAutoReUploading(isAutoReUploading => {
        console.log('isAutoReUploading：', isAutoReUploading)
        this.isAutoReUploading = isAutoReUploading
      })
      // 调用文件上传逻辑
      uploader
        .uploadFile()
        .then(response => {
          console.log(response)
          this.$message({ type: 'success', message: response.message })
        })
        .catch(error => {
          console.error(error)
          this.$message.error('文件上传失败')
        })
    },
    handleRemove(options) {
      console.log('options', options)
      delateFileByPath(options.filePath, this.saveType).then(res => {
        if (res.success) {
          this.$message({ type: 'success', message: '删除成功' })
          this.fileList = this.fileList.filter(item => item.name !== options.fileName)
        } else {
          this.$message.error('删除失败')
        }
      })
    }
  }
}
</script>

Attributes ​

参数说明类型描述可选值默认值
showFileList是否显示已上传文件列表Boolean—true
uploadPercentage文件上传进度Number—0
fileList上传文件列表Array—[]
isUploading是否正在上传Boolean—false
isAutoReUploading是否超时自动重传中Boolean—false

Events ​

事件名说明参数
start-upload点击【点击上传】触发选中的文件信息
handle-remove点击 上传文件列表中的【X】触发选中的文件基本信息

分片上传逻辑层 ​

TrFileShardingUpload类将分片上传逻辑进行封装，对外暴露必要的属性值和钩子函数，以支持不同的UI。

基本用法：

js
const uploader = new TrFileShardingUpload(file, businessType, saveType)
uploader.onProgress(progress => {
    // 处理进度更新事件
    console.log('上传进度：', progress + '%')
})

uploader.onFileInfo(fileInfo => {
    // 处理文件列表更新事件
    console.log('文件信息已更新：', fileInfo)
})

uploader.onIsUploading(isUploading => {
    console.log('isUploading：', isUploading)
})

uploader.onIsAutoReUploading(isAutoReUploading => {
    console.log('isAutoReUploading：', isAutoReUploading)
})
// 调用文件上传逻辑
uploader
    .uploadFile()
    .then(response => {
    console.log(response)
})
    .catch(error => {
    console.error(error)
})
