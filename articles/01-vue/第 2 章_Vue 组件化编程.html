**基本使用：**创建组件=>注册组件=>使用组件

html
<div id="app">
    {{msg}}
    <hr/>
    <!-- 3.使用组件 -->

<script type="text/javascript">
    //1.创建组件
    const school = Vue.extend({
        name:'school',
        template:`<div>
            <h2>学校名称：{{school.name}}</h2>
            <h2>学校地址：{{school.address}}</h2>
            </div>`,

            school:{
                name:'麻省理工学院',
                address:'111'

    //1.创建组件

        name:'student',
        template:`<div>
            <h2>学生名称：{{student.name}}</h2>
            <h2>学生地址：{{student.address}}</h2>
            </div>`,
        data(){
            return{
                student:{
                    name:'Tom',
                    address:'米国'
                }
            }
        }
    })
    new Vue({
        el:'#app',
        data:{
          msg:'hello'
        },
        //2.局部注册组件
        components:{
            school,
            student
        }
    })
</script>

如何定义一个组件?

使用 Vue.extend(options)创建，其中 options 和 new Vue(options)时传入的那个 options 几乎一样，但也有点区别

区别如下:

1.el 不要写，为什么? 最终所有的组件都要经过一个 vm 的管理，由 vm 中的 el 决定服务哪个容器

2.data 必须写成函数，为什么? 避免组件被复用时，数据存在引用关系。

备注:使用 template 可以配置组件结构。

如何注册组件?

1.局部注册: 靠 new Vue 的时候传入 components 选项

2.全局注册: 靠 Vue.component('组件名',组件)

三、编写组件标签:

<school></schoo1>

1.6. VueComponent 构造函数 ​

关于 VueComponent:

1.school 组件本质是一个名为 VueComponent 的构造函数，且不是程序员定义的，是 Vue.extend 生成的.

2.我们只需要写或，Vue 解析时会帮我们创建 school 组件的实例对象，即 Vue 帮我们执行的: new VueComponent(options)。

3.特别注意: 每次调用 Vue.extend，返回的都是一个全新的 VueComponent! ! !!

4.关下 this 指向:

(1).组件配置中:data 函数、methods 中的函数、watch 中的函数、computed 中的函数 它们的 this 均是 VueComponent 实例对象,

(2).new Vue()配置中:

data 函数、methods 中的函数、watch 中的函数、computed 中的函数 它们的 this 均是 Vue 实例对象。

5.VueComponent 的实例对象，以后简称 vc(也可称之为: 组件实例对象)。Vue 的实例对象，以后简称 vm。

6.VueComponent 不能配置 el；最终所有的组件都要经过一个 vm 的管理，由 vm 中的 el 决定服务哪个容器；

分析 Vue 和 VueComponent 的关系：

1.一个重要的内置关系:
vueComponent.prototype._proto === Vue.prototype
2.为什么要有这个关系: 让组件实例对象(vc) 可以访问到 Vue原型上的属性、方法
1
2
3
js
prototype 通常是构造函数所拥有的属性，它用于定义构造函数创建的对象的共享属性和方法。
而 __proto__ 是每个对象都拥有的属性，它提供了访问对象原型的一种方式。

prototype 是函数对象特有的一个属性，它指向一个对象，这个对象是在调用构造函数创建实例时，新对象的原型。
通过在原型对象上定义的属性和方法可以被新创建的对象共享。||||||||
通常情况下，当你创建一个函数时，它会自动拥有一个名为 prototype 的属性。例如：

function Person(name) {
  this.name = name;
}

Person.prototype.greet = function() {
  return `Hello, ${this.name}!`;
};

const person1 = new Person('Alice');
console.log(person1.greet()); // Output: Hello, Alice!

__proto__ 是每个 JavaScript 对象（实例）都具有的属性，它指向创建该对象的构造函数的原型。它提供了访问对象的原型的一种方式，实际上是访问了对象的隐式原型。例如：

const obj = {};
console.log(obj.__proto__ === Object.prototype); // Output: true
