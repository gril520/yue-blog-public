1.1. 介绍 ​

Vue CLI 是一个基于 Vue.js 进行快速开发的完整系统，提供：

  • 通过 @vue/cli 实现的交互式的项目脚手架。
  • 通过 @vue/cli + @vue/cli-service-global 实现的零配置原型开发。
  • 一个运行时依赖 (@vue/cli-service)，该依赖：
    • 可升级；
    • 基于 webpack 构建，并带有合理的默认配置；
    • 可以通过项目内的配置文件进行配置；
    • 可以通过插件进行扩展。
  • 一个丰富的官方插件集合，集成了前端生态中最好的工具。
  • 一套完全图形化的创建和管理 Vue.js 项目的用户界面。

Vue CLI 致力于将 Vue 生态中的工具基础标准化。它确保了各种构建工具能够基于智能的默认配置即可平稳衔接，这样你可以专注在撰写应用上，而不必花好几天去纠结配置的问题。与此同时，它也为每个工具提供了调整配置的灵活性，无需 eject。

1.2. 关于不同版本的Vue ​

  • vue.js与vue.runtime.xxx.js的区别

    1. vue.js是完整版的Vue, 包含：核心功能 + 模板解析器

Mixins 简介和使用

在Vue.js中，mixins（混入）是一种用于封装可重用功能的一种方式。Mixins 允许你在多个组件之间共享代码，类似于多继承。一个 mixin 对象可以包含任何组件选项。当组件使用 mixin 时，所有 mixin 中的选项将被合并到组件自身的选项中。

以下是一个简单的例子，演示了如何创建和使用一个 mixin：

js
// 定义一个名为 `exampleMixin` 的 mixin 对象
const exampleMixin = {
  data() {
    return {
      message: 'Hello from mixin!',
    };
  },
  methods: {
    logMessage() {
      console.log(this.message);
    },
  },
};

// 在组件中使用 mixin
export default {
  mixins: [exampleMixin],
  data() {
    return {
      // 组件自身的数据
      additionalData: 'Additional data in component',
    };
  },
  mounted() {
    // 可以访问来自 mixin 的数据和方法
    this.logMessage(); // 输出 "Hello from mixin!"
  },
};

在上面的例子中，exampleMixin 定义了一个包含 data 和 methods 的 mixin。然后，我们在组件中使用了这个 mixin，通过 mixins 选项传递了一个包含 exampleMixin 的数组。这样，组件将继承 exampleMixin 中定义的数据和方法。

需要注意的是，如果 mixin 和组件都有相同名称的选项（例如，data、methods 等），它们将会被合并，但在发生冲突时，组件的选项将会覆盖 mixin 的选项。

尽管 mixins 提供了一种方便的代码共享机制，但过度使用 mixins 也可能导致代码复杂性和不可预测性增加。因此，在使用 mixins 时，建议谨慎设计和组织你的代码，以避免潜在的问题。

mixins执行时机

在Vue.js中，mixins 的执行时机是在组件实例创建时。具体来说，当组件实例被创建时，Vue.js 会按照一定的顺序合并（混合）mixins中的选项到组件的选项中。这个过程包括数据、生命周期钩子、方法等。

下面是 mixins 的执行过程：

  1. 混入选项合并： Vue.js 将组件的选项与 mixins 中的选项进行合并。如果有冲突，组件的选项将覆盖 mixin 中的选项。
  2. 生命周期钩子执行： 在组件生命周期的相应时机，执行混入中的生命周期钩子。生命周期钩子按照混入的顺序被调用。
  3. 数据合并： Vue.js 会将组件的数据和 mixins 中的数据进行合并。如果有冲突，组件的数据将覆盖 mixin 中的数据。
  4. 方法合并： Vue.js 会将组件的方法和 mixins 中的方法进行合并。如果有冲突，组件的方法将覆盖 mixin 中的方法。

需要注意的是，当多个 mixins 中具有相同生命周期钩子、数据、方法等时，它们将按照数组的顺序依次执行或合并。如果有多个 mixins 中有相同的属性，Vue.js 会按照 mixin 的顺序进行合并。

以下是一个简单的示例，演示了 mixins 的执行时机：

js
const myMixin1 = {
  created() {
    console.log('Mixin 1 created');
  },
};

const myMixin2 = {
  created() {
    console.log('Mixin 2 created');
  },
};

export default {
  mixins: [myMixin1, myMixin2],
  created() {
    console.log('Component created');
  },
};

<template>

    <div class="todo-wrap">
      <my-header :submitTodo="submitTodo" />
      <my-content :todoData="todoData" :deleteTodo="deleteTodo" />

<script>
import { nanoid } from "nanoid";
import MyContent from "./MyContent.vue";
import MyFooter from "./MyFooter.vue";
import MyHeader from "./MyHeader.vue";

export default {
  name: "TodoList",
  components: {
    MyHeader,
    MyContent,
    MyFooter,
  },
  data() {
    return {
      todoData: [
        { id: "01", title: "吃饭", done: false },
        { id: "02", title: "睡觉", done: true },
        { id: "03", title: "打游戏", done: false },
      ],
    };
  },
  methods: {
    //新增
    submitTodo(e) {
      if (!e.target.value.trim()) return alert("输入不能为空"); 
        //trim()方法不会改变原字符串的值,而是返回一个新的字符串。
      const todoObj = { id: nanoid(), title: e.target.value, done: false };
      this.todoData.unshift(todoObj);
      e.target.value = "";
    },
    //删除
    deleteTodo(id) {
      if (confirm("你确定删除吗？")) {
        this.todoData = this.todoData.filter((item) => item.id !== id);
      } 
    },
    //全选
    checkAllTodo(value) {
      this.todoData.map((item) => (item.done = value));
    },
    //删除所有已完成任务
    deleteAllDone() {
      this.todoData = this.todoData.filter((item) => !item.done);
    },
  },
};
</script>

1.一种组件间通信的方式，适用于: 子组件 ==> 父组件

1.16. nextTick应用 ​

简单了解：

1.语法: this.$nextTick(回调函数)

2.作用: 在下一次 DOM 更新结束后执行其指定的回调

3.什么时候用:当改变数据后，要基于更新后的新DOM进行某些操作时，要在nextTick所指定的回调函数中执行

详细探究：

在 Vue 中，DOM 更新是异步执行的，这意味着当我们修改了数据后，DOM 并不会立即更新，而是在下一个 tick（更新周期）时才会进行更新。Vue 提供了 nextTick 方法来处理在 DOM 更新完成后执行回调函数的需求。nextTick 方法可以将回调函数推入一个回调队列，在下一个 tick 时执行这些回调函数。

在 Vue 实例上调用 $nextTick 方法，或者使用 Vue.nextTick 方法，都可以将回调函数添加到 nextTick 的队列中。

js
// 在 Vue 实例内部使用
this.$nextTick(() => {
  // DOM 更新后执行的回调函数
});

// 使用全局方法
Vue.nextTick(() => {
  // DOM 更新后执行的回调函数
});

1.22 vue中provide与inject ​

在 Vue.js 中，provide 和 inject 是一对用于父组件向子组件传递数据的高级选项。

这对选项使得祖先组件可以向嵌套的后代组件传递数据，而不需要通过中间的组件传递。

provide 选项 ​

provide 是一个在祖先组件中声明的选项，用于提供数据。它的值可以是一个对象或一个返回对象的函数。该对象中的属性和方法会被传递给所有的后代组件。

js
// 祖先组件
export default {
  provide() {
    return {
      message: 'Hello from ancestor!',
      // 其他提供的数据...
    };
  },
};

inject 选项 ​

inject 是一个在后代组件中声明的选项，用于接收祖先组件通过 provide 提供的数据。inject 的值可以是一个数组，其中包含需要注入的属性名，也可以是一个对象，其中的键是本地的绑定名，而值是在 provide 选项中注入的属性名。

js
// 后代组件
export default {
  inject: ['message'],
  // 或者
  inject: {
    customMessage: 'message',
    // 其他注入的数据...
  },
  mounted() {
    console.log(this.message); // 访问来自祖先的数据
  },
};
