1.1. vue-router 简介 ​

vue-router 是 vue 的一个插件库，专门用来实现 SPA 应用

对 SPA 应用的理解

  1. 单页 Web 应用 single page web application，SPA。

  2. 整个应用只有一个完整的页面。

  3. 点击页面中的导航链接不会刷新页面，只会做页面的局部更新。

  4. 数据需要通过 ajax 请求获取。

多页面通过在不同的 html 页面跳转，那么在单页面中要如何实现这种跳转功能呢？

路由就是用于实现在单页面中的跳转功能

路由的概念：

一个路由就是一组映射关系 key-value

key 为路径, value 可能是 function 或 component

1.5. router-link replace 属性 ​

浏览器历史记录具有 push 和 replace 两种模式，默认为 push，其是一种栈结构；而 replace 是将新的路由地址直接替换之前的路由地址；

<router-link>的 replace 属性 1.作用:控制路由跳转时操作浏览器历史记录的模式

2.浏览器的历史记录有两种写入方式:分别为 push 和 replace，push 是追加历史记录，replace 是替换当前记录。路由跳转时候默认为 push

3.如何开启 replace 模式:

html
<router-link replace .......>News</router-link>
//完整写法
<router-link :replace="true" .......>News</router-link>

    console.log('@@@：deactivated')
    clearInterval(this.timer)
  }
}

3.全局路由守卫

需求：

只有浏览器缓存 localStorage 中缓存了 user 属性，且 user 的 value 值为 yue 才可以查看 Routing hook 导航栏下的内容；

效果：

js
//全局前置路由守卫一-初始化的时候被调用、每次路由切换之前被调用  meta可用于储存一些额外的配置属性
router.beforeEach((to, from, next) => {
  console.log(to, from)
  if (to.meta.isAuth) {
    if (localStorage.getItem('user') == 'yue') {
      next()
    } else {
      alert('无权限访问')
    }
  } else {
    next()
  }
})
//全局后置路由守卫一-初始化的时候被调用、每次路由切换之后被调用
router.afterEach((to, from) => {
  console.log(to, from)
  document.title = to.name || 'router'
})
export default router

1.10 路由器的两种工作模式 ​

1.对于一个 ur 来说，什么是 hash 值?-

#及其后面的内容就是 hash 值

2.hash 值不会包含在 HTTP 请求中，即: hash 值不会带给服务器

3.hash 模式:

1.地址中永远带着#号，不美观。

2.若以后将地址通过第三方手机 app 分享，若 app 校验严格，则地址会被标记为不合法.

3.兼容性较好

4.history 模式

1.地址干净，美观

2.兼容性和 hash 模式相比略差

3.应用部署上线时需要后端人员支持，解决刷新页面服务端 404 的问题 【mark 一下 2023.11.18 日 刷新不报 404 错误了】

const router = new VueRouter({
  mode: history, //开启history路由模式
  route:[...]
 })
