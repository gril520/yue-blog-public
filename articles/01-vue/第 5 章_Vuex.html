第 5 章：vuex ​

1.介绍 ​

概念：

专门在 Vue 中实现集中式状态（数据）管理的一个 Vue 插件，对 vue 应用中多个组件的共享状态进行集中式的管理（读/写），也是一种组件间通信的方式，且适用于任意组件间通信。

什么时候使用 Vuex：

  1. 多个组件依赖于同一状态

  2. 来自不同组件的行为需要变更同一状态

对比全局总线：

双箭头，取与修改；全局事件总线一条线只能取；要同时实现取和改比较麻烦；

2.vuex 原理图以及核心概念、API ​

state，mutations 和 actions 都是对象，都由 store 进行管理；

vuex 核心概念和 API

state   vuex 管理的状态对象 它应该是唯一的

actions
1.值为一个对象，包含多个响应用户动作的回调函数；
2.通过 commit()来触发 mutation 中函数的调用, 间接更新 state
3.触发 actions 中的回调：$store.dispatch('对应的 action 回调名')
4.可以包含异步代码（定时器, ajax 等等）

mutations
1. 值是一个对象，包含多个直接更新 state 的方法
2.在 action 中使用或者代码中直接使用：commit('对应的 mutations 方法名') 触发
3. mutations 中方法的特点：不能写异步代码、只能单纯的操作 state

/*
 * @Descripttion:用于创建vuex中最为核心的store
 * @Version: 1.0
 * @Date: 2023-11-16 13:41:07
 */
import Vue from 'vue'
//导入vuex插件
import Vuex from 'vuex'
//应用Vuex插件
Vue.use(Vuex)

//准备action 用于响应组件中的动作
const actions = {
  odd(context, value) {
    console.log('odd', context, value)
    if (context.state.sum % 2 != 0) {
      context.commit('ODD', value)
    }
  },
  delay(context, value) {
    console.log('delay', context, value)
    setTimeout(() => {
      context.commit('ODD', value)
    }, 1000)
  }
}
//准备mutations 用于操作数据（state）
const mutations = {
  JIA(state, value) {
    console.log('JIA', state, value)
    state.sum += value
  },
  JIAN(state, value) {
    state.sum -= value
  },
  ODD(state, value) {
    state.sum += value
  }
}
//准备state 用于存储数据
const state = {
  sum: 99
}
//创建并且暴露store
export default new Vuex.Store({
  actions,
  mutations,
  state
})

1.使用 mapState 映射状态取数据, mapState 用于帮助我们映射 state 中的属性为计算属性；

<template>
  <div class="container">
    <h3>一、this.$store.state方式取数据</h3>

      <li v-for="item in list" :key="item.id">{{ item.name }}-----{{ item.age }}</li>
    </ul>
    <hr />
    <h3>二、使用mapState映射状态取数据</h3>
    <h4>学校地址：{{ address }}</h4>
    <h4>学生人数：{{ num }}</h4>
    <h4>老师人数：{{ teacherNum }}</h4>
    <h3>二、使用mapGetters映射状态取数据</h3>
    <hr />
    <h4>放大十倍结果为：{{ bigSum }}</h4>
    <h4>放大十倍结果为：{{ getBigSum }}</h4>
  </div>

<script>
import { mapGetters, mapState } from 'vuex'
export default {
  name: 'MySchool',
  data() {
    return {}
  },
  computed: {
    schoolName() {
      return this.$store.state.schoolName
    },
    list() {
      return this.$store.state.teacherList
    },
    // 数组方式
    ...mapState(['address', 'num']),
    //对象方式
    ...mapState({ teacherNum: 'tnum' }),
    // 数组方式
    ...mapGetters(['bigSum']),
    //对象方式
    ...mapGetters({ getBigSum: 'bigSum' })
  }
}
</script>

vue
<!--
 * @Descripttion: 
 * @Version: 1.0

<template>
  <div class="container" :style="{ color: 'red', marginTop: '40px' }">
    <input v-model="name" type="text" @keydown.enter="add" /><button @click="addLi">添加姓李的人员</button>
    <h3>求和结果为：{{ $store.state.countAbout.sum }}</h3>
    <ul>
      <li v-for="item in $store.state.personAbout.personList" :key="item.id">{{ item.name }}</li>
    </ul>
    <h3>Person人数为：{{ getLength }}</h3>
    <h3>ajax请求数据:</h3>
    <input v-model="keyValue" />
    <button @click="addAjax">发送请求</button>
    <ul>
      <li v-for="item in $store.state.personAbout.ajaxDatas" :key="item.url">{{ item.login }}</li>
    </ul>
  </div>
</template>

<script>
import { nanoid } from 'nanoid'
export default {
  name: 'MyPerson',
  data() {
    return {
      name: '',
      keyValue: ''
    }
  },
  computed: {
    getLength() {
      //这个写法很奇特
      return this.$store.getters['personAbout/getLength']
    }
  },
  methods: {
    add() {
      const obg = { id: nanoid(), name: this.name }
      this.$store.commit('personAbout/ADD_PERSON', obg)
      this.name = ''
    },
    addLi() {
      const obg = { id: nanoid(), name: this.name }
      this.$store.dispatch('personAbout/addLi', obg)
      this.name = ''
    },
    addAjax() {
      this.$store.dispatch('personAbout/requestData', this.keyValue)
    }
  }
}
</script>

    requestData(context, value) {

          console.log('请求失败了', error.message)
        }
      )
    }
  },
  mutations: {
    ADD_PERSON(state, value) {
      state.personList.unshift(value)
    },
    AJAX_DATA(state, value) {
      state.ajaxDatas = value.splice(0, 5)
    }
  },
  state: {
    personList: [{ id: '001', name: '张三' }],
    ajaxDatas: []
  },
  getters: {
    getLength(state) {
      return state.personList.length
    }
  }
}
export default personOptions

MyPersonPlus.vue 注意看取数据和写数据

取数据需要加 user

但写数据可以直接 commit 和 dispatch

思考：这是一种偷懒的写法，缺点是这样是的结构不清晰，但会减小在模块化时操作 vuex 中数据的写法；

vue
<!--
 * @Descripttion: 
 * @Version: 1.0
 * @Date: 2023-11-13 18:44:10
-->
<template>
  <div class="container" :style="{ color: 'red', marginTop: '40px' }">
    <input v-model="name" type="text" @keydown.enter="add" /><button @click="addLi">添加姓李的人员</button>
    <h3>求和结果为：{{ $store.state.countAbout.sum }}</h3>
    <ul>
      <li v-for="item in $store.state.user.personList" :key="item.id">{{ item.name }}</li>
    </ul>
    <h3>Person人数为：{{ getLength }}</h3>
  </div>
</template>

<script>
import { nanoid } from 'nanoid'
export default {
  name: 'MyPersonPlus',
  data() {
    return {
      name: '',
      keyValue: ''
    }
  },
  computed: {
    getLength() {
      //这个写法很奇特
      return this.$store.getters['getLength']
    }
  },
  methods: {
    add() {
      const obg = { id: nanoid(), name: this.name }
      console.log('@@@', this.$store)
      this.$store.commit('ADD_PERSON', obg)
      this.name = ''
    },
    addLi() {
      const obg = { id: nanoid(), name: this.name }
      this.$store.dispatch('addLi', obg)
      this.name = ''
    }
  }
}
</script>
